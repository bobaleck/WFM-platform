# Этап 9.8. Предпродовая доводка WFM

Дата: 2026-06-02.

## 1С-сверка сотрудника

WFM больше не считает `/health` Gateway сверкой сотрудника. Кнопка "Сверить с 1С" вызывает `/api/v1/1c/check-employee-status` и передаёт параметры базы 1С из сохранённых backend-настроек: тип базы, сервер, имя информационной базы или путь файловой базы, пользователя, защищённо сохранённый пароль и ИНН сотрудника. Секреты не выводятся в логи и отчёты.

В карточке сотрудника показывается только итоговый кадровый статус: "Работает", "Уволен", "Не найден в 1С", "Ошибка сверки", "Не проверялся". Технические поля количества активных/уволенных карточек и стратегия lookup скрыты из пользовательской карточки и оставлены только для диагностики интеграции.

## Сотрудники

Список сотрудников разделён на "Активные" и "Архив". Вся строка сотрудника кликабельна и раскрывает подробную карточку. Кнопка "Открыть" удалена; действия вынесены в меню: редактировать, сверить с 1С, сверить с Naumen, уволить в WFM, архивировать.

Редактирование вынесено в отдельный экран сотрудника с кнопками "Назад", "Сохранить" и "Отмена". При выходе с несохранёнными изменениями показывается подтверждение. В форме есть проекты/контуры, команда активного проекта, глобальные навыки, UUID Naumen и основные поля сотрудника. СНИЛС и логин Naumen не используются.

Архивирование меняет только локальный WFM-статус. В архиве можно восстановить сотрудника или удалить окончательно из WFM. Окончательное удаление разрешено только для архивных сотрудников без связанных графиков или истории; 1С и Naumen при этом не изменяются.

## Команды и навыки

Раздел команд теперь показывает только текущий состав команды. Добавление участников вынесено в отдельное окно с поиском по ФИО и ИНН. Backend проверяет, что сотрудник доступен в проекте команды.

Навыки остаются глобальными. В разделе "Навыки" добавлено назначение навыка сотрудникам через модальное окно с поиском. В раскрытии навыка видны сотрудники с этим навыком и доступно действие "Убрать навык".

## Naumen

Активный рабочий проект "X5 Retail Group" связан с UUID Naumen `corebo00000000000p2hcq728jesuk0o`. Неактивная дубль-запись проекта сохранена и не используется как рабочий контур.

В интеграции Naumen добавлены действия: проверить подключение, загрузить операторов проекта, сопоставить сотрудников по ФИО. Загрузка очередей и нагрузки из Naumen оставлена disabled, потому что endpoint интервальной статистики/нагрузки не подтверждён текущей документацией. Fake-данные не создавались.

## Нагрузка, потребность, графики и отсутствия

Раздел "Нагрузка" описывает интервальную статистику контакт-центра: поступило, обработано, потеряно, AHT и SL/SLA по очередям. Доступны шаблон и ручная загрузка файла. Загрузка из Naumen отображается как неподтверждённая.

Раздел "Потребность" объясняет расчёт операторов по интервалам через нагрузку, AHT, целевую занятость и резерв. Английские подписи target occupancy/shrinkage заменены на русские пользовательские тексты.

Графики показывают workflow: сотрудники, команды, смены, нагрузка, потребность, черновик, покрытие, подтверждение и публикация. Видимые статусы назначений и рекомендаций переведены на русский.

Отсутствия больше не требуют ручного ввода ID сотрудника. Сотрудник выбирается из списка активного проекта с поиском по ФИО/ИНН. Типы и статусы отсутствий отображаются по-русски.

## Отчёты

Отчёты сохранены по вкладкам: руководитель, операции, покрытие, план/факт, SLA, экспорты. Важность рекомендаций отображается на русском. Выгрузки выполняются через авторизованный fetch.
