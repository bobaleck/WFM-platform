# WFM MVP Scope

Этап 2 переводит проект из skeleton в рабочую внутреннюю MVP-систему WFM.

## Реализовано

- Справочник сотрудников.
- Справочник команд.
- Справочник навыков.
- Справочник очередей.
- Интервальная нагрузка.
- Расчётная потребность в операторах как хранимые записи.
- Справочник смен.
- Назначения графиков.
- Отсутствия.
- KPI-сводка.
- Демо-данные для проверки интерфейса и API.
- Информационная заглушка интеграции.
- CSV-импорт интервальной нагрузки.
- Расчёт потребности операторов.
- Правила планирования.
- Черновое формирование графиков.
- Очереди и требуемые навыки.
- Генерация графиков с учётом обязательных навыков.
- Покрытие по интервалам.
- Подтверждение и публикация графиков.
- План/факт.
- CSV-экспорт графиков, покрытия и план/факт.
- Авторизация пользователей.
- Роли и права.
- Журнал действий.
- Импорт фактической работы из CSV.
- Журнал экспортов.
- Базовый application hardening.

## Не входит в текущий MVP

- Реальная интеграция с Naumen или другой телефонией.
- Промышленный алгоритм расчёта графиков.
- Сложный прогноз нагрузки.
- Alembic migrations.
- Production TLS.
- SSO.
- Production backup и мониторинг.

## Следующий объём

Этап 6: миграции Alembic, резервное копирование, мониторинг, production deployment и подготовка адаптера внешних данных без реального подключения.

## Этап 5

Этап 5 добавил внутреннюю авторизацию, роли, права, журнал действий, импорт фактической работы, журнал экспортов и базовые меры hardening. Внешние источники данных по-прежнему не подключаются.

Роли MVP: `admin`, `supervisor`, `analyst`, `customer`, `readonly`.

Импорт факта принимает CSV с датой, email сотрудника, очередью, интервалом, статусом и фактическими минутами. Ошибки строк сохраняются в `import_errors`.
# Обновление Этапа 7

MVP включает executive dashboard, управленческие отчёты, coverage gaps, SLA summary, план/факт и рекомендации планировщика. Планировщик остаётся MVP scoring algorithm без подключения Timefold.
