# Naumen integration

Модуль подготовлен по локальному PDF `/home/alex/ncc_rest_api_ru.pdf`.

На этапе 7.2 реализованы:

- хранение настроек подключения без открытого возврата секретов;
- построение заголовков Basic и API key;
- подтверждённые методы сотрудников `/api/v2/employees` и `/api/v2/employees/{login}`;
- pending-методы для отделов, операторов проекта, расписания проекта и аудиозаписей, потому что соответствующие endpoints не были надёжно подтверждены в извлечённом тексте PDF;
- dry run API без внешних сетевых запросов.

Реальные запросы к Naumen должны включаться отдельным этапом после проверки тестового контура и актуальной документации.
