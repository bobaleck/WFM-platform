## Правила модели

WFM остаётся основной карточкой сотрудника. 1С используется только для кадрового статуса по ИНН. Naumen используется для операторского UUID, операторов проекта и будущей статистики.

Сотрудник отображается в контуре, если есть запись `employee_project_access`. Команда принадлежит одному проекту, а добавить в неё можно только сотрудника с доступом к этому проекту. Навыки глобальны и могут назначаться любому сотруднику.

## Naumen по ФИО

При сверке WFM берёт ФИО сотрудника, нормализует регистр, пробелы и букву `ё`, затем ищет оператора среди локально загруженных операторов текущего проекта Naumen. Если есть отчество, требуется точное совпадение фамилии, имени и отчества. Если отчества нет, используется фамилия и имя.

Если найден один кандидат, сохраняется `naumen_uuid`. Если найдено несколько, статус становится «Есть расхождения» и требуется ручной выбор. Если ФИО неполное или проект Naumen не настроен, возвращается понятная ошибка.
