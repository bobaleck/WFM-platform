# Отчёт Этапа 9.1: ручной WFM MVP

Дата: 2026-06-01.

## 1. Исходное состояние

Перед изменениями выполнен backup БД: `/opt/wfm-naumen/backups/db/wfm_naumen_20260601T072754Z.sql`.

Проверки показали, что backend, frontend, scheduler, PostgreSQL и Redis запущены. Health backend возвращает `{"status":"ok","service":"backend"}`. `/api/v1/version` после доработки сообщает `external_source=manual` и `onec_integration=gateway_http`.

Проблемы до этапа:

- Naumen оставался в рабочем интерфейсе и API;
- часть CRUD-кнопок была нерабочей или неполной;
- логотип и favicon не отдавались из frontend-контейнера;
- контуры были привязаны к старой модели проектов;
- настройки были техническими и мало применимыми;
- нагрузка, потребность и пустые состояния были недостаточно понятны.

## 2. Что исправлено

- Naumen удалён из рабочего UI, а активный `/api/v1/integrations/naumen/*` больше не регистрируется.
- Раздел «Интеграции» содержит только 1С.
- Логотип скопирован в `frontend/public/assets/cvet_tss.svg`, favicon — в `frontend/public/favicon.svg`; Dockerfile frontend теперь копирует `public`.
- Сайдбар показывает рабочие контуры, а не партнёров Naumen.
- Добавлен API контуров: `/api/v1/contours`.
- Сотрудники получили действия: создать, редактировать, сверить с 1С, уволить, вернуть, архивировать.
- Команды, навыки, очереди, смены и отсутствия получили рабочий CRUD на уровне MVP.
- Добавлены шаблон и импорт нагрузки XLSX/CSV.
- Потребность рассчитывается с дефолтными настройками без `undefined`.
- Настройки приложения стали редактируемыми через `/api/v1/settings/app`.
- Отсутствия получили иконку перечёркнутого календаря.

## 3. Текущий WFM-процесс

Рабочий сценарий: создать контур, заполнить сотрудников, команды, навыки, очереди и смены, загрузить нагрузку, рассчитать потребность, сформировать черновик графика, проверить покрытие, подтвердить и опубликовать график. Статус сотрудника сверяется с 1С по ИНН вручную или массово.

## 4. 1С

1С остаётся единственной рабочей внешней интеграцией. На Linux Direct COM не запускается: требуется отдельный Windows 1C Gateway с установленной платформой 1С и COMConnector.

Правило статуса:

- если найдено физлицо и есть хотя бы одна карточка без даты увольнения — «Работает»;
- если все найденные карточки имеют дату увольнения — «Уволен»;
- если ИНН не найден — «Не найден»;
- при ошибке подключения — «Ошибка сверки».

Текущая диагностика без Gateway возвращает понятное состояние: адрес Windows 1C Gateway не заполнен.

## 6. Ограничения

- Production Windows 1C Gateway ещё не реализован.
- Direct COM на текущем Linux-сервере недоступен.
- Erlang C пока не реализован.
- Планировщик графиков остаётся MVP-алгоритмом и требует проверки управляющим.
- В коде остаются исторические Naumen-модули и документы, но они не доступны из рабочего UI/API.
