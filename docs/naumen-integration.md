# Интеграция Naumen

Naumen возвращён как отдельный источник операторских данных и статистики контакт-центра. Он не является кадровым источником увольнений и не заменяет WFM-карточку сотрудника.

## Назначение

- UUID оператора Naumen;
- оператор в проекте/рабочем контуре;
- локальная загрузка операторов проекта;
- сопоставление операторов Naumen с WFM-сотрудниками;
- будущая загрузка статистики, AHT, SLA, обращений и фактических выходов после уточнения endpoint-ов.

## Настройки

Настройки Naumen находятся в разделе «Интеграции» отдельно от 1С:

- Base URL;
- API version `v2` или `current`;
- auth mode;
- username;
- API token или Basic password;
- verify SSL;
- timeout;
- enabled.

Секреты хранятся зашифрованно и не возвращаются открытым текстом.

## Операторы проекта

Локальная таблица `naumen_operators` хранит операторов, загруженных по проекту:

- `project_id`;
- `naumen_uuid`;
- `project_uuid`;
- `partner_uuid`;
- ФИО, email, phone;
- state/substate;
- skills/raw_data;
- last_seen_at.

Оператор Naumen не становится WFM-сотрудником автоматически. Администратор или управляющий сопоставляет его с сотрудником по ФИО внутри проекта; UUID сохраняется только после уверенного выбора.

## Сопоставление

Порядок сопоставления:

1. Если `naumen_uuid` уже сохранён, WFM подтверждает связь с локальным оператором проекта.
2. Если UUID нет, WFM ищет оператора по нормализованному ФИО внутри проекта Naumen.
3. Если найден один кандидат, UUID сохраняется.
4. Если найдено несколько кандидатов, требуется ручной выбор.

Naumen не меняет ИНН и не меняет статус 1С.

## Ограничения

Endpoint интервальной статистики и статистики сотрудника пока ожидает уточнения. WFM не создаёт fake-данные: если статистика не загружена, карточка сотрудника показывает пустое состояние.
# Актуализация этапа 9.4

Naumen возвращён как операционный источник, а не кадровая система.

Использование Naumen:

- загрузка операторов выбранного проекта;
- хранение локального справочника `naumen_operators`;
- сопоставление сотрудника WFM с оператором Naumen по ФИО внутри проекта;
- сохранение `naumen_uuid` только при единственном уверенном совпадении;
- будущая загрузка фактических выходов и статистики, когда endpoint будет подтверждён.

Логин Naumen больше не используется в пользовательском интерфейсе и шаблонах. Сверка выполняется по ФИО:

- фамилия + имя + отчество, если отчество заполнено;
- фамилия + имя, если отчества нет;
- при нескольких кандидатах WFM не привязывает оператора автоматически и требует ручной выбор.

Naumen не меняет ИНН и не определяет увольнение сотрудника.
# Актуализация этапа 9.7

Naumen-интеграция используется для операторов проекта и статистики контакт-центра. Первичная сверка сотрудника теперь выполняется по ФИО внутри активного WFM-проекта/проекта Naumen:

1. WFM загружает или использует локально загруженных операторов проекта.
2. Сначала ищется полное ФИО.
3. Если полное ФИО не найдено, используется фамилия + имя.
4. Если найден один кандидат, сохраняется `naumen_uuid`.
5. Если найдено несколько кандидатов, WFM требует ручной UUID.
6. UUID проверяется отдельным действием «Проверить UUID».

Логин Naumen не используется. Naumen не определяет кадровый статус и не заменяет данные 1С.

По локальной документации подтверждён сценарий операторов проекта. Endpoints статистики сотрудника, AHT/SLA/обращений и фактических выходов требуют дополнительного подтверждения; WFM не создаёт fake-метрики.

# Актуализация Naumen/NCC

Статистика Naumen/NCC теперь берётся через read-only PostgreSQL NCC только backend-ом WFM. Frontend никогда не подключается к NCC напрямую.

Обязательные env без секретов в коде:

- `NCC_DB_HOST`;
- `NCC_DB_NAME`;
- `NCC_DB_USER`;
- `NCC_DB_PASSWORD`;
- `NCC_DB_PORT`;
- `NCC_DB_TIMEOUT_SECONDS`;
- `NCC_DB_MAX_PERIOD_DAYS`.

Фильтрация клиента выполняется по `partneruuid`, а не по названию партнёра. Активные проекты NCC выбираются только со `state = 'Активный'` и `removed = false`.

Если env или UUID контура не заполнены, WFM показывает понятное состояние “Интеграция Naumen/NCC не настроена” или “Naumen-статистика недоступна, используйте ручную загрузку”. Fake-метрики не создаются.
