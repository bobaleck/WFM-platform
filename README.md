# WFM-платформа Телесейлз Сервис

Self-hosted MVP внутренней WFM-системы для контактного центра. Проект развёрнут в `/opt/wfm-naumen` и работает через Docker Compose.

## Текущий статус

С Этапа 9.1 платформа переведена в ручной WFM-контур:

- Naumen возвращён как отдельный источник операторов и статистики, но не как кадровый источник;
- 1С осталась единственной внешней интеграцией для сверки статуса сотрудника по ИНН;
- сотрудники создаются вручную или загружаются XLSX-реестром;
- команды, навыки, очереди, смены, отсутствия и графики ведутся вручную;
- нагрузка загружается через XLSX/CSV;
- потребность рассчитывается из нагрузки по MVP-формуле;
- графики формируются как черновик и затем подтверждаются/публикуются;
- настройки приложения редактируются в UI;
- логотип и favicon отдаются из frontend public assets.

## Запуск

```bash
cd /opt/wfm-naumen
docker compose up -d --build
```

## Проверка

```bash
docker compose ps
curl -s http://127.0.0.1/health
curl -s http://127.0.0.1/api/v1/version
npm --prefix frontend run build
docker compose exec -T backend pytest
```

## Вход

- URL: `http://<server>/login`
- Пользователь MVP: `admin`
- Пароль берётся из `ADMIN_PASSWORD` в `/opt/wfm-naumen/.env`

Пароли, токены, `JWT_SECRET` и строки подключения нельзя выводить в логи, отчёты и командный вывод.

## Основной процесс

1. Создать рабочий контур.
2. Создать команды, навыки, очереди и смены.
3. Создать сотрудников вручную или загрузить реестр XLSX.
4. Сверить сотрудников с 1С по ИНН.
5. При необходимости привязать UUID Naumen и загрузить операторов проекта.
6. Загрузить нагрузку XLSX/CSV.
7. Рассчитать потребность.
8. Сформировать черновик графика.
9. Проверить покрытие.
10. Подтвердить и опубликовать график.
11. Проверить отчёты и выгрузки.

## Интеграция с 1С

Текущая архитектура:

```text
WFM backend -> HTTP -> Windows 1C Gateway -> COMConnector -> база 1С
```

Direct COM на Linux недоступен. Для production нужен отдельный Windows Gateway.

## Интеграция Naumen/NCC

Операторы, очереди, нагрузка и статистика Naumen/NCC читаются backend-ом из read-only PostgreSQL NCC. Frontend не подключается к NCC напрямую.

В `.env` backend заполняются только реальные значения, в `.env.example` хранятся пустые шаблоны:

```env
NCC_DB_HOST=
NCC_DB_NAME=
NCC_DB_USER=
NCC_DB_PASSWORD=
NCC_DB_PORT=5432
NCC_DB_TIMEOUT_SECONDS=15
NCC_DB_MAX_PERIOD_DAYS=31
```

Если env или UUID Naumen/NCC у контура не заполнены, интерфейс показывает ручной режим: нагрузка загружается XLSX/CSV, fake-данные не создаются.

## Ограничения MVP

- production Windows 1C Gateway ещё не реализован;
- Erlang C пока не реализован;
- графики формируются MVP-алгоритмом и требуют проверки управляющим;
- реальные endpoint-ы Naumen для статистики сотрудника и интервальной нагрузки требуют уточнения.
# Актуализация WFM MVP, этап 9.4

Платформа использует WFM как основную карточку сотрудника. Сотрудники могут быть доступны в нескольких рабочих контурах, команды принадлежат конкретному контуру, навыки глобальные. 1С используется для кадровой сверки по ИНН через Windows Gateway, Naumen используется для операторов проекта, UUID и будущей статистики.

В интерфейсе сотрудников СНИЛС и логин Naumen не используются. Сопоставление с Naumen выполняется по ФИО внутри выбранного проекта Naumen.
# Этап 9.7

WFM-side стабилизация 1С и Naumen выполнена на сервере `192.168.12.31`.

- 1С Gateway работает на отдельном Windows Server `10.1.1.12` и обновляется пользователем вручную.
- WFM не использует fake `ok` для сотрудников.
- Ответ «Физлицо найдено, но карточки сотрудников не найдены» отображается как диагностическая проблема Gateway.
- Подготовлен пакет `/opt/wfm-naumen/tools/onec-gateway` для ручного копирования на Windows.
- Naumen-сопоставление сотрудника первично выполняется по ФИО, UUID используется как fallback.
# Этап 9.8

WFM работает как ручная платформа: сотрудники, команды, глобальные навыки, очереди, нагрузка, потребность, смены, графики, отсутствия и отчёты. 1С сверяет кадровый статус по ИНН через Windows Gateway. Naumen используется для операторов проекта X5 Retail Group и будущей статистики, если endpoint подтверждён.
