# Модель идентификации сотрудника

WFM является основной системой карточки сотрудника. Внешние системы дают отдельные идентификаторы:

- 1С: ИНН и кадровый статус;
- Naumen: UUID оператора и операционная статистика.

## WFM

Ключевые поля сотрудника:

- `project_id` / рабочий контур;
- ФИО;
- ИНН;
- СНИЛС;
- контакты;
- должность;
- команда;
- навыки;
- WFM-статус;
- комментарий.

Сотрудник всегда относится к одному рабочему контуру. Списки сотрудников фильтруются по текущему контуру.

## 1С

ИНН используется только для сверки с 1С. Допустимые статусы:

- `not_checked`;
- `working`;
- `dismissed`;
- `not_found`;
- `check_error`;
- `gateway_unavailable`;
- `gateway_invalid`;
- `gateway_timeout`;
- `onec_connection_error`.

Статус `ok` запрещён для поля `onec_status`.

## Naumen

UUID Naumen используется только для сопоставления с оператором Naumen. Допустимые статусы:

- `not_linked` — «Не сопоставлен»;
- `linked` — «Сопоставлен»;
- `not_found` — «Не найден в Naumen»;
- `mismatch` — «Есть расхождения»;
- `check_error` — «Ошибка сверки»;
- `api_unavailable` — «Naumen недоступен»;
- `no_access` — «Нет доступа».

Один `naumen_uuid` не должен быть привязан к двум активным сотрудникам одного контура.
# Актуализация этапа 9.4

Основная карточка сотрудника ведётся в WFM. Сотрудник больше не считается принадлежащим только одному проекту: доступ к рабочим контурам хранится в `employee_project_access`.

Пользовательская модель:

- ФИО, ИНН, дата рождения, email, телефон, должность, статус WFM и комментарий;
- проекты/контуры через флаги доступа;
- команда в рамках конкретного проекта через `employee_team_memberships`;
- глобальные навыки через `employee_skills`;
- статус 1С по ИНН;
- UUID и статус Naumen по сопоставлению оператора.

СНИЛС и логин Naumen считаются legacy-полями: физически могут оставаться nullable в БД для совместимости, но не показываются в интерфейсе, не входят в XLSX-шаблон и не используются в новых пользовательских сценариях.

1С определяет только кадровый статус по ИНН. Naumen не является кадровой системой: он используется для операторского UUID, операторов проекта, выходов и статистики.
# Актуализация этапа 9.7

Карточка сотрудника WFM хранит:

- ИНН для 1С;
- `naumen_uuid` для Naumen;
- статусы и сообщения последней сверки 1С;
- статусы и сообщения последней сверки Naumen;
- `onec_metadata` с диагностикой Gateway.

Статус `ok` запрещён для сотрудника. Общий статус `connected/ok` допустим только для интеграции в целом. Если 1С Gateway нашёл физлицо, но не нашёл карточки сотрудников, карточка получает диагностический статус «Карточки сотрудника не найдены».

Naumen первично сопоставляется по ФИО. UUID используется как ручной fallback и проверяется отдельной кнопкой.
# Обновление этапа 9.8

Карточка сотрудника WFM является основной. ИНН используется для 1С, UUID Naumen — для операторского сопоставления. Пользовательская карточка скрывает технические поля Gateway и показывает только человекочитаемые статусы. Сотрудник может быть архивирован локально в WFM без изменения 1С и Naumen.
