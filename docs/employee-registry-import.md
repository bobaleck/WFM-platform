# Импорт реестра сотрудников

Сотрудники создаются вручную или загружаются XLSX-реестром.

## API

- шаблон: `GET /api/v1/employees/import/template.xlsx`;
- импорт: `POST /api/v1/employees/import/xlsx`.

## Шаблон

Файл `.xlsx` содержит лист «Сотрудники»:

- ФИО;
- ИНН;
- Дата рождения;
- Email;
- Телефон;
- Должность;
- Команда;
- Навыки;
- Проекты/контуры;
- UUID Naumen;
- Комментарий.

Второй лист «Справка» содержит описание колонок, пример ИНН и пример навыков через запятую.

## Правила импорта

- максимум 10 000 строк;
- ФИО обязательно;
- ИНН обязателен для сверки с 1С и должен содержать 12 цифр;
- сотрудник создаётся или обновляется по ИНН и получает доступы к указанным проектам/контурам;
- дубли по ИНН не создаются;
- команда создаётся автоматически, если включено `auto_create_teams`;
- навыки глобальные и создаются автоматически, если включено `auto_create_skills`;
- СНИЛС и логин Naumen не используются в шаблоне;
- ошибки строк пишутся в `employee_import_errors`;
- общий результат пишется в `employee_import_batches`.

## Результат

Backend возвращает:

- `rows_total`;
- `rows_created`;
- `rows_updated`;
- `rows_failed`;
- `errors`.

Frontend показывает результат импорта и таблицу ошибок, если они есть.
