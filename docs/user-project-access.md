# Доступ пользователей к проектам

На Этапе 8 добавлено ограничение видимости WFM-данных по проектам Naumen.

## Правила

- `admin` и `is_superuser` видят все активные проекты.
- Пользователь с обычной ролью видит только проекты, назначенные в `user_project_access`.
- Если у пользователя нет доступных проектов, UI показывает пустое состояние: «Нет доступных проектов. Обратитесь к администратору».
- Если доступен один проект, sidebar показывает его как активный.
- Если доступно несколько проектов, пользователь выбирает проект в sidebar.

## Таблицы

`user_project_access`:

- `user_id`;
- `naumen_project_id`;
- `can_view`;
- `can_sync`.

`user_preferences`:

- `user_id`;
- `selected_project_id`.

## Создание пользователя

Random user больше не создаётся.
Кнопка «Создать пользователя» открывает модальную форму:

- ФИО;
- username;
- email;
- роль;
- активность;
- временный пароль и подтверждение;
- чекбоксы доступных проектов.

Пароль не логируется и не сохраняется открытым текстом.
Нельзя отключить или удалить последнего администратора.

## Фильтрация данных

List endpoints основных WFM-разделов принимают `project_id` и проверяют доступ пользователя.
Если пользователь запрашивает проект без доступа, backend возвращает `403`.

Проектный фильтр добавлен для:

- сотрудников;
- команд;
- очередей;
- смен;
- нагрузки;
- потребности;
- графиков;
- покрытия;
- отсутствий;
- KPI;
- факта.
