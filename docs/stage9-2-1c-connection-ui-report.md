# Отчёт Этапа 9.2: интерфейс подключения к 1С

Дата: 2026-06-01.

## 1. Что было неправильно

Старая форма 1С содержала поле «Адрес Gateway» без пояснения и поля «База 1С»/«Путь к файловой базе» без контекста. Это могло выглядеть так, будто от пользователя требуется Web-ссылка на опубликованную базу 1С.

## 2. Почему база 1С не публикуется в Web

База 1С остаётся внутри локальной сети. WFM не требует и не предполагает Web-публикацию базы 1С. На Linux WFM не может использовать COMConnector напрямую, поэтому подключение выполняет отдельный внутренний Windows Gateway.

## 3. Почему нужен Windows Gateway

COMConnector доступен на Windows-хосте с установленной платформой 1С. Текущий WFM-сервер работает на Linux, поэтому правильная схема:

```text
WFM backend -> внутренний Windows Gateway -> COMConnector -> база 1С
```

## 4. Какие поля используются

Форма теперь разделена на блоки:

- Windows Gateway: внутренний адрес Gateway, token Gateway, TLS, timeout;
- параметры базы 1С: тип базы, сервер 1С, имя информационной базы, кластер, путь к файловой базе 1С, пользователь и пароль 1С;
- поведение сверки: включение интеграции, сверка при создании, автоотключение уволенных, еженедельная сверка.

## 5. Как сохраняются секреты

Пароль 1С и token Gateway хранятся в backend зашифрованно. GET settings возвращает только `password_saved` и `gateway_token_saved`. Пустой пароль или token в PUT не затирает сохранённое значение.

## 6. Проверка подключения

`POST /api/v1/integrations/onec/check` валидирует настройки и при заполненных параметрах вызывает внутренний Gateway:

```text
POST {gateway_url}/api/v1/1c/check-connection
```

В тело запроса передаются параметры базы 1С. Пароль не логируется и не возвращается в ответе WFM.

## 7. Диагностика

`POST /api/v1/integrations/onec/diagnose` возвращает ОС сервера, режим подключения, безопасный адрес Gateway, признаки заполненности параметров, доступность Direct COM и список недостающих полей. Пароль, token и строка подключения не возвращаются.

## 8. Что сделать для реального подключения

1. Установить Windows Gateway на Windows-сервер с платформой 1С.
2. Разрешить доступ к Gateway только от WFM-сервера.
3. Указать внутренний адрес Gateway.
4. Указать тип базы, сервер 1С и имя информационной базы или путь к файловой базе.
5. Указать пользователя и пароль 1С.
6. Выполнить проверку подключения.
